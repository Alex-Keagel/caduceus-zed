## Findings

### 🔴 Critical
1. [crates/language_model/src/registry.rs:322] `has_authenticated_provider()` bypasses `cached_auth_state()`, violating AC-PERF1 and forcing repeated provider reads in fan-out paths. **Fix:** Replace the body with `self.providers.keys().any(|id| self.cached_auth_state(id, cx).can_provide_models())`.
2. [crates/language_model/src/registry.rs:328] `available_models()` bypasses `cached_auth_state()`, so panel/selector renders still pay full auth-state fan-out cost. **Fix:** Replace the filter with `.filter(|provider| self.cached_auth_state(&provider.id(), cx).can_provide_models())`.
3. [crates/language_models/src/provider/bedrock.rs:505] Bedrock maps unauthenticated state to `EnterApiKeyInSettings`, which will dispatch users into the wrong ST1b UX. **Fix:** Replace `action: AuthAction::EnterApiKeyInSettings,` with `action: AuthAction::None,` until a Bedrock-specific AWS credential action exists.
4. [crates/language_model/src/auth_state.rs:47] `DisabledByPolicy { reason: String }` is directly constructible, so S3 sanitization is optional rather than enforced. **Fix:** Replace the variant with `DisabledByPolicy { reason: SanitizedProviderReason }` and introduce a newtype `pub struct SanitizedProviderReason(String);` with a private field plus `as_str()` accessor; keep `ProviderAuthState::disabled_by_policy(...)` as the only constructor.
5. [crates/language_model/src/registry.rs:425] `note_completion_error()` pre-clamps `retry_after`, so `ProviderAuthState::rate_limited()` never logs oversized upstream values on this path. **Fix:** Replace lines 425-427 with:
   ```rust
   let state = ProviderAuthState::rate_limited(*retry_after, AuthAction::None);
   let deadline = retry_after
       .map(|d| d.min(crate::MAX_RETRY_AFTER))
       .map(|d| Instant::now() + d);
   ```
6. [missing: provider round-trip test] The stop-ship T8 `provider_auth_state_round_trip` coverage across all 16 providers is absent. **Fix:** Add a parametric test table that asserts every provider's authenticated and unauthenticated mapping, including Bedrock=`None`, Cloud=`SignInImperative`, Copilot disabled=`DisabledByPolicy`, and local providers=`None`.
7. [missing: CI shim gate] AC5/T18 is not enforced anywhere in CI, so deprecated `is_authenticated(` usage can regress silently. **Fix:** Add `script/ci/check-auth-shim-annotations.sh` that fails on `rg 'is_authenticated\(' crates --glob '*.rs'` lines lacking `auth-shim: usability-check`, and wire it into CI.

### 🟡 Important
1. [crates/language_model/src/registry.rs:355] The docstring says fan-out callers “should go through this method,” but the registry itself violates that contract. **Fix:** After fixing the methods above, update the comment to `All registry fan-out helpers (`available_models`, `has_authenticated_provider`, `configuration_error`) route through this cache.`
2. [crates/language_model/src/auth_state.rs:72] The safe constructor exists, but the API shape does not force callers to use it. **Fix:** Add `#[must_use]` to `disabled_by_policy` and the sanitized newtype accessor to make the intended path explicit.
3. [crates/language_model/src/language_model.rs:303] The deprecated shim remains without any automated guardrail. **Fix:** Add a doc comment line: `// Every remaining call site must carry // auth-shim: usability-check and is enforced by script/ci/check-auth-shim-annotations.sh.` once the gate exists.

### 🟢 Suggestions
1. [crates/language_model/src/auth_state.rs:20] The comment says over-cap retry-after is “silently clamped + logged”; “silently” is false if the warning fires. **Fix:** Replace with `this is clamped to MAX_RETRY_AFTER and logged at WARN`.
2. [crates/language_model/src/registry.rs:407] Avoid re-encoding clamp semantics in two places. **Fix:** Keep clamping/logging solely in `ProviderAuthState::rate_limited()` and treat the registry as deadline derivation only.

## Recommended Actions
1. Fix the two cache-bypass call sites first; they violate the plan and the registry's own contract.
2. Fix Bedrock's unauth action before ST1b lands, or users will be routed into the wrong remediation flow.
3. Enforce S3 in the type system; advisory sanitization is not a security control.
4. Restore retry-after warning visibility on the completion-error path.
5. Add T8 and AC5/T18 before merge; both are explicit stop-ship items.
