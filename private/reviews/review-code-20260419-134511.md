## Findings

### 🔴 Critical
1. [crates/agent/src/thread.rs:3870] — Spawn edges use `entity_id()` for the child instead of the child thread `SessionId`, so parent→child DAG edges are not in the same identity space as the actual thread/session graph. **Fix:** Change `register_running_subagent` to accept `cx: &App`, upgrade the weak entity, and record `subagent.read(cx).id().0.to_string()`.
2. [crates/agent_ui/src/agent_panel.rs:4390,4850] — "Live auto-refresh" is false. The DAG only refreshes when render happens; an idle panel stays stale forever. **Fix:** Move refresh scheduling to a persistent poll task started in `new()` and stop triggering refresh from `render_top_bar`.
3. [crates/caduceus_bridge/src/index_dag.rs:300] — Read/read sharing is labeled as contention, which is a false conflict signal. **Fix:** Require at least one writer before a resource becomes a contention point.
4. [crates/agent/src/agent.rs:1671] — `/help` omits `/dag` even though autocomplete exposes it. **Fix:** Add `/dag` to the help text or generate help from the same command table.
5. [design gap] — There is no dedicated dockable DAG panel/view. A badge tooltip is not the requested feature. **Fix:** Either implement a separate `DagPanel` or change all user-facing wording to say this is only a badge tooltip plus `/dag`.

### 🟡 Important
1. [crates/caduceus_bridge/src/index_dag.rs:269] — Edge aggregation is O(n²) because every event linearly searches `edges`. **Fix:** Aggregate with a `HashMap<(DagNode, DagNode, AccessKind), u32>`.
2. [tests] — Missing regression tests for idle refresh, `/help` parity, and two-readers-no-contention. **Fix:** Add those tests before claiming the feature is complete.

### 🟢 Suggestions
1. [crates/agent_ui/src/agent_panel.rs:4782] — `⛓ {spawns}/{accesses}` is not self-explanatory. **Fix:** Use explicit labels like `⛓ S:{spawns} A:{accesses}` or a dedicated panel header.

## Recommended Actions
1. Fix identity correctness first (`SessionId`, not `entity_id`).
2. Add a real timer/poller so idle UI still refreshes.
3. Correct contention semantics and add tests.
4. Either ship a real dockable DAG panel or stop claiming one exists.
